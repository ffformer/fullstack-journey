02 — 变量提升（Hoisting）与暂时性死区（TDZ）

一、什么是变量提升？

JS 引擎在执行代码之前，会先把所有 var 声明和函数声明提到当前作用域的顶部。这个过程叫"提升"。

注意：只提升声明，不提升赋值。

示例：var 的提升

console.log(a);  // undefined（不是报错！因为 var 声明被提升了）
var a = 10;
console.log(a);  // 10

// 上面代码等价于：
// var a;          ← 声明被提升到顶部
// console.log(a); → undefined（声明了但还没赋值）
// a = 10;         ← 赋值留在原地
// console.log(a); → 10

二、函数的提升

function 声明的函数会整体提升（声明 + 函数体一起提升），可以在声明前调用。

示例：函数提升 vs 函数表达式不提升

sayHi(); // ✅ "Hello!" — 函数声明整体提升

function sayHi() {
  console.log('Hello!');
}

sayBye(); // ❌ TypeError: sayBye is not a function
var sayBye = function() {
  console.log('Bye!');
};
// var sayBye 被提升 = undefined，undefined() 报错

三、let 和 const 的"暂时性死区"（TDZ）

let 和 const 也会提升，但提升后的行为不同：

  • var 提升后初始化为 undefined，可以访问
  • let/const 提升后不初始化，从块开始到实际声明行之间的区域称为"暂时性死区"（Temporal Dead Zone）
  • 在 TDZ 中访问变量会报 ReferenceError

示例：TDZ 演示

// === TDZ 开始 ===
console.log(name); // ❌ ReferenceError: Cannot access 'name' before initialization
// === TDZ 结束 ===
let name = '张三';
console.log(name); // ✅ "张三"

// typeof 在 TDZ 中也不安全：
console.log(typeof x); // ❌ ReferenceError（TDZ 内的 let x）
let x = 1;

四、var / let / const 对比总结

特性 var let const
作用域 函数作用域 块级作用域 块级作用域
提升后初始值 undefined 未初始化（TDZ） 未初始化（TDZ）
可重复声明 ✅ 可以 ❌ 不可以 ❌ 不可以
可重新赋值 ✅ 可以 ✅ 可以 ❌ 不可以
声明前访问 undefined 报错（TDZ） 报错（TDZ）

五、实战：变量提升造成的经典 bug

示例：if 条件中 var 导致的迷惑行为

var count = 100;

if (true) {
  console.log(count); // 你以为是 100？实际是 undefined！
  var count = 1;      // 这个 var 被提升到函数/全局顶部，覆盖了外部的访问
  console.log(count); // 1
}

// 用 let 就不会有这个问题：
let total = 100;
if (true) {
  // console.log(total); ← 如果取消注释会报错 TDZ
  let total = 1;  // 这是块内的新变量，不影响外部的 total
  console.log(total); // 1
}
