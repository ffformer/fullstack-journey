DOM 事件层级 · 事件流 · 事件委托

理解事件的本质：从 HTML 属性到 addEventListener，从捕获到冒泡

一、DOM 事件级别

JS 事件绑定方式经历了从"写在 HTML 里"到"addEventListener"的演变

1. DOM Level 0：HTML 属性绑定

最早的方式——直接在 HTML 标签上写事件，或给 DOM 属性赋值

// 方式一：HTML 内联（不推荐——耦合太强）
<button onclick="alert('hello')">点击</button>
// 方式二：DOM 属性赋值（缺点在后面）
btn.onclick = function() {
  console.log('点击了')
}
// 方式三：通过 setAttribute（不推荐）
btn.setAttribute('onclick', 'fn()')

2. DOM Level 0 的致命缺陷

同一个事件只能绑定一个处理函数，后者会覆盖前者

// 第一个处理函数
btn.onclick = () => console.log('处理函数 A')
// 第二个处理函数 → 把 A 覆盖了！
btn.onclick = () => console.log('处理函数 B')
→ 点击后只输出"处理函数 B"，A 丢了!
// 解除绑定也只能设为 null
btn.onclick = null

3. DOM Level 2：addEventListener

可以绑定多个处理函数，还能控制捕获/冒泡阶段

// 绑定事件 — 不会覆盖
btn.addEventListener('click', handlerA)
btn.addEventListener('click', handlerB)
→ 点击后依次执行 handlerA、handlerB
// 解除绑定 — 必须用同一个函数引用
btn.removeEventListener('click', handlerA)
// ⚠️ 匿名函数无法移除！

4. DOM Level 3：更多事件类型

Level 3 主要扩展了事件种类，没有改变事件绑定机制

// Level 2 已有（核心事件）
click, keydown, submit, focus, blur, load
mouseover, mouseout, mousedown, mouseup
// Level 3 新增（更丰富的交互事件）
'wheel' — 滚轮事件（替代 mousewheel）
'input' — 输入实时触发（替代 keypress 场景）
'transitionend' — CSS 过渡结束
'animationend' — CSS 动画结束
'pointerdown' — 指针按下（统一鼠标+触摸）

二、事件流（Event Flow）

事件在 DOM 树中的传播路径：捕获阶段 → 目标阶段 → 冒泡阶段

1. 事件的三个阶段

每次事件触发，都经历三个阶段的传播

① 捕获阶段（Capture）
window → document → html → body → ... → 目标父元素
② 目标阶段（Target）
到达实际触发事件的元素
③ 冒泡阶段（Bubble）
目标父元素 → ... → body → html → document → window

2. 捕获 vs 冒泡（addEventListener 第三参数）

默认在冒泡阶段触发；设 true 则在捕获阶段触发

// 默认：冒泡阶段触发（第三个参数 false/省略）
el.addEventListener('click', fn)
el.addEventListener('click', fn, false)
// 捕获阶段触发
el.addEventListener('click', fn, true)
// 或写成对象形式（推荐）
el.addEventListener('click', fn, { capture: true })

3. 事件冒泡（Bubbling）详解

事件从最深的元素向上传播，每一层父元素都能监听到

// HTML 结构：
// <div class="parent">
// <button class="child">点我</button>
// </div>
// 点击 button 时，click 事件会冒泡：
// button → div.parent → body → html → document
child.addEventListener('click', () => console.log('子元素'))
parent.addEventListener('click', () => console.log('父元素也触发!'))

4. stopPropagation — 阻止传播

阻止事件继续向上冒泡，但不阻止同一元素上的其他监听器

child.addEventListener('click', (e) => {
  e.stopPropagation()
  console.log('只在子元素触发，父元素收不到')
})
// stopImmediatePropagation 更彻底：
// 阻止冒泡 + 阻止同一元素上后续监听器的执行
el.addEventListener('click', (e) => {
  e.stopImmediatePropagation()
})

5. preventDefault — 阻止默认行为

阻止浏览器默认行为，但不阻止事件传播（重点！）

// 阻止 a 标签跳转
link.addEventListener('click', (e) => {
  e.preventDefault() // a 不会跳转
  // 但事件仍会冒泡到父元素！
})
// 常见场景：
// - 阻止表单提交刷新页面
// - 阻止右键菜单弹出
// - 阻止 checkbox 被勾选

6. 哪些事件不冒泡？

大部分事件冒泡，但有一批常用事件不冒泡

// ❌ 不冒泡的事件：
focus, blur, load, unload, error
mouseenter, mouseleave
DOMContentLoaded
// ✅ 冒泡版本：
focus → focusin（冒泡）
blur → focusout（冒泡）
mouseenter → mouseover（冒泡）
mouseleave → mouseout（冒泡）

四、事件委托（Event Delegation）

利用事件冒泡，在父元素上统一监听——DOM 事件最重要的实战技巧之一

1. 为什么要事件委托？

动态添加的元素无法预先绑定事件，事件委托解决这个问题

// ❌ 问题：新添加的元素没有事件
document.querySelectorAll('li').forEach(li => {
  li.addEventListener('click', handler)
})
// 后来 append 的 li 没有事件！
// ✅ 事件委托：在父元素统一监听
ul.addEventListener('click', (e) => {
  if (e.target.matches('li')) handler(e)
})
→ 动态添加的 li 也能响应!

2. 事件委托的三个关键点

理解 target vs currentTarget，以及如何精确匹配

// e.target — 实际点击的元素（最深层）
// e.currentTarget — 绑定事件的元素（即 this）
// 1. matches() — 精确匹配选择器
if (e.target.matches('.btn')) { ... }
// 2. closest() — 向上查找最近的匹配元素
const btn = e.target.closest('.btn')
if (btn) { ... } // 点击 .btn 内部子元素也能匹配
// 3. contains() — 判断是否包含某元素
if (el.contains(e.target)) { ... }

3. 事件委托实战模板

几乎所有动态列表都用这个模式

// 通用委托模板
container.addEventListener('click', (e) => {
  const target = e.target.closest('.item')
  if (!target) return // 不是我们关心的元素
  // 通过 data-* 获取数据
  const id = target.dataset.id
  console.log('点击了:', id)
})

5. stopPropagation 对委托的影响

在委托元素内部的某个子元素上调 stopPropagation 会导致委托失效

// ❌ 危险：内部停止冒泡，委托监听器收不到
innerBtn.addEventListener('click', (e) => {
  e.stopPropagation() // 父元素的委托失效！
})
// ✅ 解决：把委托设在捕获阶段
parent.addEventListener('click', handler, true)

五、addEventListener 第三个参数（高级选项）

1. once — 只执行一次

触发一次后自动移除，无需手动 removeEventListener

el.addEventListener('click', () => {
  console.log('只触发一次!')
}, { once: true })
→ 第二次点击不会有任何反应
// 实战：一次性加载动画结束回调
el.addEventListener('animationend', cleanup, { once: true })

2. passive — 不调用 preventDefault

告诉浏览器"我不会阻止默认行为"，提升滚动性能

// 移动端滚动优化关键！
el.addEventListener('touchstart', handler, { passive: true })
el.addEventListener('wheel', handler, { passive: true })
// ⚠️ passive 和 preventDefault 不能共存
// 设了 passive: true 再调 e.preventDefault() → 无效+警告
// Chrome 默认将 touchstart/wheel 设为 passive

3. signal — AbortController 批量移除

一次性移除多个事件监听器，比逐个 removeEventListener 优雅

const controller = new AbortController()
el.addEventListener('click', fn1, { signal: controller.signal })
el.addEventListener('input', fn2, { signal: controller.signal })
// 一键全部移除！
controller.abort()
// 实战：组件销毁时清理所有事件
class Component {
  constructor() {
    this.controller = new AbortController()
  }
  destroy() { this.controller.abort() }
}

4. 自定义事件 CustomEvent

创建自己的事件类型，解耦模块间通信

// 创建自定义事件
const event = new CustomEvent('user:login', {
  detail: { id: 1, name: '张三' }
})
el.dispatchEvent(event)
// 监听自定义事件
el.addEventListener('user:login', (e) => {
  console.log('登录用户:', e.detail.name)
})
