Promise · 异步编程

从"为什么需要异步"到"Promise 原理与实战"——一条线讲清

一、同步 vs 异步：JavaScript 的执行模型

1. 同步（Synchronous）— 一行等一行的阻塞执行

代码一行接一行执行，上一步没做完，下一步就不开始

console.log('① 开始煮饭')
// 假设这行要等 3 秒（烧水）
for (let i = 0; i < 3e9; i++) {} // 忙等 3 秒
console.log('② 饭煮好了')
console.log('③ 开始炒菜') // 必须等煮饭完成!
→ ①②③ 依次输出，②到③之间什么都做不了

2. 异步（Asynchronous）— 非阻塞，先干别的

耗时的操作丢给浏览器线程，代码继续往下执行，完成后再通知你

console.log('① 打开电磁炉煮水')
setTimeout(() => {
  console.log('③ 水开了！（3秒后通知）')
}, 3000)
console.log('② 趁水没开，先切菜')
→ ①②（同时做）... 3秒后 ... ③

3. 为什么必须用异步？

JS 是单线程的——一次只能做一件事。如果同步等网络请求，页面就卡死了

// 如果网络请求是同步的：
// 用户点击按钮 → 发请求 → 等 2 秒 → 才能继续操作页面
// 这 2 秒内，页面完全冻结：点击无效、滚动无效、动画卡死
// JS 的解决方式：
// 把耗时任务委托给浏览器（浏览器有多线程）
// 主线程继续执行，等任务完成再"回调"通知
// → 这就是"异步"的本质

4. 常见的异步操作

JS 中这些操作都是异步的

// 定时器
setTimeout(fn, 1000)
setInterval(fn, 1000)
// 网络请求（最重要的场景）
fetch('/api/user')
// 读文件（Node.js）
fs.readFile('/data.json', callback)
// 事件监听
btn.addEventListener('click', fn)
// DOM 渲染相关
requestAnimationFrame(fn)

二、问题：回调地狱（Callback Hell）

Promise 诞生之前，异步靠回调函数——当多层异步依赖时，代码变成"金字塔"

5. 回调地狱长这样 👇

每个异步操作的结果依赖上一个结果，层层嵌套

// 需求：登录 → 获取用户信息 → 获取订单 → 获取详情
login(user, password, (err, token) => { // ① 登录
  if (err) { /* 处理错误 */ }
  getUser(token, (err, user) => { // ② 查用户
    if (err) {}
    getOrders(user.id, (err, orders) => { // ③ 查订单
      if (err) {}
      getDetail(orders[0].id, (err, detail) => { // ④ 查详情
        if (err) {}
        console.log('终于拿到数据:', detail)
      })
    })
  })
}) // 😱 箭头越来越深 —— "回调地狱"

6. 回调地狱的三大问题

不只是不好看，更是维护和可靠性的噩梦

① 可读性差 — 深层嵌套，"doom pyramid"
② 错误处理重复 — 每一层都要 if(err)，大量重复代码
③ 无法统一控制 — "等 A 和 B 都完成再做 C" 用回调很难实现
✅ Promise 解决这三个问题
— 用 .then() 链式调用替代嵌套
— 一个 .catch() 处理所有错误
— Promise.all 等组合方法处理并发

三、Promise 是什么？—— 一个"承诺"

7. 生活类比

Promise 就像外卖订单，它有三种状态

// 你下单外卖 →
// [等待中] ：商家正在做，你不知道最终能不能送到
// [已送达] ：外卖到了，你开吃（成功回调）
// [已取消] ：商家取消订单，你退款（失败回调）
// 状态一旦从 [等待中] 变成其他，就再也不会变

8. Promise 的三种状态核心

这正是 Promise 的基本原理——状态机

pending
进行中
→
fulfilled
已成功
或
rejected
已失败
// 规则：
// ① 状态只能从 pending → fulfilled 或 pending → rejected
// ② 状态一旦改变就不可逆——这叫"决议"（resolve）
// ③ fulfilled 和 rejected 统称 settled（已敲定）

9. 创建 Promise核心

new Promise 接收一个 executor 函数，里面做异步工作

const promise = new Promise((resolve, reject) => {
  // executor 函数：在这里做异步操作
  // resolve(value) — 成功后调用，将状态改为 fulfilled
  // reject(reason) — 失败后调用，将状态改为 rejected
// 实战：把 setTimeout 包装成 Promise
const delay = ms => new Promise(resolve => {
  setTimeout(resolve, ms)
})
delay(2000).then(() => console.log('2秒后'))

10. then / catch / finally核心

这三个方法是 Promise 的使用入口——注册"成功/失败/最终"的回调

promise
  .then(value => {
    // 成功时执行，value = resolve() 传的值
  })
  .catch(error => {
    // 失败时执行，error = reject() 传的值
  })
  .finally(() => {
    // 无论成功与否都执行（ES2018）
  })
// ⚠️ 注意：then 也接受第二个参数作为错误回调
.then(onSuccess, onError)
// 但推荐用 .catch()，可以捕获 then 中的错误

四、链式调用：Promise 的核心威力

then 返回的也是一个 Promise → 可以无限链下去 → 解决回调地狱

11. .then() 返回新 Promise 的规则

这是理解链式调用的关键，面试高频

// .then() 始终返回一个新的 Promise
// 新 Promise 的值 = then 回调的返回值
then 回调 return 什么下一个 then 得到什么
普通值，如 4242
一个新的 Promise等这个 Promise resolve/reject
什么都没 returnundefined
throw new Error()跳到最近的 .catch()

12. 用 Promise 改写回调地狱

同样的需求，用链式调用变成清晰的"流水线"

login(user, password)
  .then(token => getUser(token))
  .then(user => getOrders(user.id))
  .then(orders => getDetail(orders[0].id))
  .then(detail => console.log('拿到数据:', detail))
  .catch(err => console.error('任何一步出错都在这处理', err))
→ 扁平化！四个操作像流水线一样清晰

13. 错误穿透机制

链中任何一步出错都会跳过中间的 then，直达最近的 catch

fetch('/api/data')
  .then(res => res.json())
  .then(data => {
    throw new Error('业务异常') // 抛出错误
  })
  .then(data => {
    console.log('永远不会到这里') // 被跳过了
  })
  .catch(err => {
    console.error('捕获到:', err) // ✅ 这步执行
  })

五、Promise 静态方法（组合多个 Promise）

14. Promise.all — 全部成功才成功

等所有 Promise 完成，只要有一个失败就整体失败

const [user, posts, settings] = await Promise.all([
  fetch('/api/user').then(r => r.json()),
  fetch('/api/posts').then(r => r.json()),
  fetch('/api/settings').then(r => r.json()),
])
→ 等 3 个请求全部完成，结果数组 [user, posts, settings]
// ⚠️ 任何一个 reject，整个 all 立即 reject
// 适合：多个独立请求，全部需要才渲染页面

15. Promise.allSettled — 全部完成就成功ES2020

等所有完成（不论成败），每个结果带有 status 和 value/reason

const results = await Promise.allSettled([
  fetch('/api/ok'),
  fetch('/api/broken'), // 这个会 500
])
// → [{status:"fulfilled",value:...}, {status:"rejected",reason:...}]
// 不会 reject，适合"能拿多少拿多少"的场景

16. Promise.race — 第一个完成的决定结果

多个 Promise 竞赛，取最快完成的（不管成功失败）

// 请求超时控制（经典用法）
Promise.race([
  fetch('/api/data'),
  new Promise((_, reject) =>
    setTimeout(() => reject('超时!'), 5000)
  )
])
→ 5 秒内没拿到数据就 reject('超时!')

17. Promise.any — 第一个成功的决定结果ES2021

取第一个成功的，只有全部失败才 reject

// 从多个 CDN 加载资源，哪个快用哪个
Promise.any([
  fetch('https://cdn1.com/file.js'),
  fetch('https://cdn2.com/file.js'),
  fetch('https://cdn3.com/file.js'),
])
→ 三个 CDN 只要有一个成功就 OK

六、async / await：Promise 的语法糖（ES2017）

让异步代码看起来像同步代码一样直白——但底层还是 Promise

18. async 函数和 await 表达式核心

async 函数自动返回 Promise；await 等待 Promise resolve

// async 函数声明
async function getUserData() {
  const user = await fetch('/api/user').then(r => r.json())
  const posts = await fetch('/api/posts').then(r => r.json())
  return { user, posts }
}
// 调用时也返回 Promise
getUserData().then(data => console.log(data))

19. async/await 错误处理

用 try/catch 捕获 await 的错误

async function safeFetch() {
  try {
    const data = await fetch('/api/data').then(r => r.json())
    return data
  } catch (err) {
    console.error('请求失败:', err)
    return null // 返回默认值
  }
}

20. .then() vs await — 什么时候用哪个

.then() 更好

并发请求（不需要等上一个结果）
Promise.all([a(), b()])

await 更好

有依赖的串行请求
先登录→再查用户→再查订单

// ❌ 慢：有依赖时用 .then 链反而啰嗦
// ✅ 快：并发时用 await 会变串行（慢），用 .all() 并行

七、Promise 原理：手写一个简易版

这是理解 Promise 原理最直接的方式——实现 then 的链式调用和状态管理

21. 手写 MyPromise（核心骨架）

实现了状态管理 + then 注册 + resolve/reject + 链式调用

class MyPromise {
  constructor(executor) {
    this.state = 'pending' // 初始状态
    this.value = undefined // resolve 的值
    this.callbacks = [] // 存放 .then() 注册的回调
    const resolve = (value) => {
      if (this.state !== 'pending') return // 状态不可逆
      this.state = 'fulfilled'
      this.value = value
      this.callbacks.forEach(cb => cb.onFulfilled(value))
    }
    const reject = (reason) => { /* 同上，state='rejected' */ }
    try { executor(resolve, reject) }
    catch (err) { reject(err) }
  }
  then(onFulfilled, onRejected) {
    return new MyPromise((resolve, reject) => {
      // 如果已 settled，直接执行；否则先存到 callbacks
    })
  }
}

22. 手写 MyPromise.all（静态方法）

理解 .all 的原理——计数 + 结果数组

static all(promises) {
  return new MyPromise((resolve, reject) => {
    const results = []
    let count = 0
    promises.forEach((p, i) => {
      p.then(value => {
        results[i] = value
        if (++count === promises.length) resolve(results)
      }, reject) // 任何一个失败 → 整体失败
    })
  })
}
