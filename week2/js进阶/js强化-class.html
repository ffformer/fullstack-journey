一、static — 类自己的方法（不属于实例）

1. static 方法和属性的本质

static 定义的东西挂在类本身，实例访问不到

class MathUtil {
  static PI = 3.14159 // 静态属性（ES2022）
  static add(a, b) { return a + b }
}
MathUtil.add(1, 2) // → 3（类上调用 ✅）
new MathUtil().add(1, 2) // ❌ TypeError
// 对比：实例方法挂在原型上
class Dog {
  bark() { return '汪汪' } // 原型方法 — 实例调用
}

2. static 的经典使用场景模式

工厂方法、缓存、单例、工具函数集合

// ① 工厂方法：根据不同情况创建实例
class User {
  constructor(data) { Object.assign(this, data) }
  static fromJson(json) {
    return new User(JSON.parse(json)) // 从 JSON 创建
  }
  static fromLocal(key) {
    const data = JSON.parse(localStorage.getItem(key))
    return new User(data) // 从 localStorage 创建
  }
}
// ② 计数/缓存
class Product {
  static #cache = new Map() // 静态私有字段
  static find(id) {
    if (!this.#cache.has(id)) {
      this.#cache.set(id, loadFromDb(id))
    }
    return this.#cache.get(id)
  }
}

3. static 初始化块（ES2022）

一次性初始化复杂静态数据，比在外面写更清晰

class Config {
  static env = {}
  static { // 静态初始化块
    try {
      this.env = JSON.parse(localStorage.getItem('config') || '{}')
    } catch {
      this.env = { debug: true }
    }
  }
}

4. 继承中的 static

子类继承父类的静态方法（静态属性也会继承）

class Base {
  static whoami() { return this.name } // this 指向调用时的类
}
class Child extends Base {}
Child.whoami()
→ "Child"（this 指向 Child，不是 Base）

二、super 详解 — 不止是调父类构造函数

5. super 的两个身份

super 在构造函数里当函数用；在方法里当对象用

// ① 构造函数中：super() = 调用父类 constructor
class Child extends Parent {
  constructor(name, age) {
    super(name) // 必须调用！且在 this 之前
    this.age = age // super 之后才能用 this
  }
}
// ② 方法中：super.method() = 调用父类同名方法
class Child extends Parent {
  greet() {
    super.greet() // 先执行父类的 greet
    console.log('再加上子类的逻辑') // 扩展
  }
}

6. super 的 this 指向

super.method() 中的 this 仍然指向子类实例

class Parent {
  getName() { return this.constructor.name }
}
class Child extends Parent {
  logName() {
    console.log(super.getName()) // 调用父方法
  }
}
new Child().logName()
→ "Child"（super.getName 中的 this 仍是 Child 实例）

7. 继承内置类（extends Array / Error）

ES6 class 可以继承原生类，这是原型链继承做不到的

// 自定义数组——给数组加上额外方法
class MyArray extends Array {
  first() { return this.at(0) }
  last() { return this.at(-1) }
  sum() { return this.reduce((s, n) => s + n, 0) }
}
const arr = MyArray.from([1, 2, 3])
arr.first() / arr.last() / arr.sum()
→ 1 / 3 / 6

8. super 在类字段中的陷阱关键

类字段（field）在 super() 之后定义，但赋值时机容易混淆

class Parent {
  constructor() { console.log('parent') }
}
class Child extends Parent {
  x = this.compute() // 类字段赋值在 super() 之后执行
  compute() { return 42 }
}
→ 顺序：parent → compute → x=42

三、Mixin — 多继承的替代方案

9. 什么是 Mixin？

JS 只能单继承，Mixin 将多个类的功能"混入"一个类

// Mixin = 一个返回 class 的函数
const TimestampMixin = Base => class extends Base {
  getTimestamp() { return new Date() }
}
// 使用：把 Mixin "套"在目标类外面
class User { /* ... */ }
class UserWithTimestamp extends TimestampMixin(User) {}
new UserWithTimestamp().getTimestamp()
→ 当前时间

10. 组合多个 Mixin

函数组合——把多个 Mixin 串起来

const pipe = (...mixins) =>
  Base => mixins.reduce((cls, mixin) => mixin(cls), Base)
// 定义两个 Mixin
const Loggable = Base => class extends Base {
  log(msg) { console.log(`[${new Date().toISOString()}] ${msg}`) }
}
const Serializable = Base => class extends Base {
  toJson() { return JSON.stringify(this) }
}
// 组合使用
class Model extends pipe(Loggable, Serializable)(Object) {}
const m = new Model()
m.log('hello') // ✅ 来自 Loggable
m.toJson() // ✅ 来自 Serializable

四、高级特性与实战模式

11. 类表达式 vs 类声明

class 可以是表达式（赋值给变量），也可以有名字

// 匿名类表达式
const MyClass = class {
  sayHi() { return 'hi' }
}
// 具名类表达式 — 名字只在类内部可见
const External = class InternalName {
  whoami() { return InternalName.name } // InternalName 内部可用
}
new External().whoami()
→ "InternalName"

12. instanceof 原理

检查对象的原型链上是否有某个类的 prototype

class Animal {}
class Dog extends Animal {}
const d = new Dog()
d instanceof Dog // → true
d instanceof Animal // → true
d instanceof Object // → true
// 手写 instanceof 原理
function myInstanceof(obj, Constructor) {
  let proto = Object.getPrototypeOf(obj)
  while (proto) {
    if (proto === Constructor.prototype) return true
    proto = Object.getPrototypeOf(proto)
  }
  return false
}

13. 私有方法 —— 不暴露内部实现

# 开头的都是私有的，外部完全不可见

class BankAccount {
  #balance = 0 // 私有字段
  deposit(amount) {
    this.#validate(amount) // 调私有方法
    this.#balance += amount
  }
  #validate(amount) { // 私有方法
    if (amount <= 0) throw new Error('金额必须大于0')
  }
}
const acc = new BankAccount()
acc.#validate(100) // ❌ SyntaxError — 不可访问

14. 单例模式模式

一个类只允许有一个实例——用静态属性控制

class AppState {
  static #instance = null
  constructor() {
    if (AppState.#instance) {
      return AppState.#instance // 返回已有的实例
    }
    AppState.#instance = this
    this.data = {}
  }
}
new AppState() === new AppState()
→ true（同一个实例）

15. 只读属性 — getter 不设 setter

只提供 get，不提供 set → 外部只能读不能写

class Rectangle {
  constructor(w, h) { this.#w = w; this.#h = h }
  get area() { return this.#w * this.#h }
  get perimeter() { return 2 * (this.#w + this.#h) }
}
const r = new Rectangle(5, 3)
r.area / r.perimeter
→ 15 / 16
r.area = 999 // ❌ 无效（没有 setter）
