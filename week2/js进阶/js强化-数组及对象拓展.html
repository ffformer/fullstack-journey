数组拓展 · 对象拓展（ES6+）

聚焦 ES6 ~ ES2024 新增的数组和对象方法 | 打开控制台（F12）看完整运行结果

一、数组拓展（ES6+ 新增方法）

1. Array.from — 类数组转数组ES6

将可迭代对象或类数组转为真正的数组，第二个参数是 map 函数

// 字符串 → 数组
Array.from('hello')
→ ["h", "e", "l", "l", "o"]
// NodeList → 数组（querySelectorAll 返回值）
const cards = document.querySelectorAll('.card')
Array.from(cards).forEach(...) // 安全遍历
// 第二个参数：生成器 + map 二合一
Array.from({ length: 5 }, (_, i) => i + 1)
→ [1, 2, 3, 4, 5]

2. Array.of — 创建数组ES6

解决 new Array(3) 的歧义——Array.of 始终用参数作元素值

// new Array() 的陷阱
new Array(3)
→ [空×3]（创建长度为 3 的空数组！）
// Array.of 没有歧义
Array.of(3)
→ [3]（包含数字 3 的数组）
Array.of('a', 1, {x:2})
→ ["a", 1, {x:2}]

3. entries / keys / values — 迭代器ES6

返回 Array Iterator，可用 for...of 或 next() 遍历

const arr = ['a', 'b', 'c']
// keys() — 索引迭代器
[...arr.keys()]
→ [0, 1, 2]
// values() — 值迭代器
[...arr.values()]
→ ["a", "b", "c"]
// entries() — [索引, 值] 迭代器
for (const [i, v] of arr.entries()) console.log(i, v)
→ 0 "a" / 1 "b" / 2 "c"

4. at() — 按索引取值ES2022

支持负数索引，从末尾倒数，比 arr[arr.length-1] 优雅

const arr = ['a', 'b', 'c', 'd']
arr.at(0) / arr.at(2)
→ "a" / "c"
arr.at(-1) // 最后一个
→ "d"
arr.at(-2) // 倒数第二个
→ "c"

5. findLast / findLastIndexES2023

从数组末尾向前查找，find/findIndex 的反向版本

const nums = [5, 12, 8, 130, 44]
nums.find(n => n > 10)
→ 12（第一个匹配）
nums.findLast(n => n > 10)
→ 44（最后一个匹配）
nums.findLastIndex(n => n > 10)
→ 4

6. 不可变数组方法（不改变原数组）ES2023

sort/reverse/splice 的不可变版本，返回新数组

const arr = [3, 1, 2]
// toSorted — sort 的不可变版
arr.toSorted()
→ [1, 2, 3]（arr 不变: [3, 1, 2]）
// toReversed — reverse 的不可变版
arr.toReversed()
→ [2, 1, 3]（arr 不变）
// toSpliced(start, deleteCount, ...items) — splice 不可变版
arr.toSpliced(1, 1, 'x')
→ [3, "x", 2]（arr 不变）
// with(index, value) — 替换单个元素
arr.with(1, 99)
→ [3, 99, 2]（arr 不变）

7. copyWithinES6

在数组内部将一段元素复制到另一位置，覆盖原值，改变原数组

// copyWithin(target, start, end)
[1,2,3,4,5].copyWithin(0, 3)
→ [4, 5, 3, 4, 5]（把索引3起的内容复制到索引0）
[1,2,3,4,5].copyWithin(1, 0, 3)
→ [1, 1, 2, 3, 5]（把0~2的内容复制到索引1起）

8. Object.groupBy — 数组分组ES2024

按条件将数组元素分组到对象中，返回普通对象（无原型）

const items = [
  { name: '苹果', type: '水果' },
  { name: '白菜', type: '蔬菜' },
  { name: '香蕉', type: '水果' },
]
Object.groupBy(items, item => item.type)
→ { 水果: [{...}, {...}], 蔬菜: [{...}] }

9. 数组解构赋值ES6

一行从数组中提取多个值，支持默认值和剩余元素

const [a, b] = [1, 2]
→ a=1, b=2
// 跳过 / 默认值 / 剩余
const [first, , third = 10, ...rest] = [1, 2, 3, 4, 5]
→ first=1, third=3, rest=[4,5]
// 交换变量（不用临时变量）
[a, b] = [b, a]

10. 展开运算符 ...ES6

在数组字面量中展开可迭代对象

// 合并数组
const merged = [...arr1, ...arr2, ...arr3]
// 浅拷贝
const copy = [...original]
// 数组去重（配合 Set）
[...new Set([1, 2, 2, 3])]
→ [1, 2, 3]

二、对象拓展（ES6+ 新增方法）

11. Object.keys / values / entriesES6/ES8

将对象转为数组再遍历——比 for...in 安全（不含原型链）

const obj = { name: '张三', age: 25 }
Object.keys(obj)
→ ["name", "age"]
Object.values(obj)
→ ["张三", 25]
Object.entries(obj)
→ [["name","张三"], ["age",25]]

12. Object.fromEntries — entries 的逆操作ES2019

将键值对数组转回对象，Map → 对象的桥梁

Object.fromEntries([
  ['name', '张三'],
  ['age', 25],
])
→ { name: "张三", age: 25 }
// 实战：Map → 对象
Object.fromEntries(map)
// 实战：URL 参数转对象
Object.fromEntries(new URLSearchParams('?a=1&b=2'))
→ { a: "1", b: "2" }

13. Object.assign — 浅拷贝/合并ES6

将多个源对象的自有可枚举属性复制到目标对象

// 合并对象
Object.assign(target, source1, source2)
// 浅拷贝（等价于 {...obj}）
const copy = Object.assign({}, original)
// ⚠️ 浅拷贝：嵌套对象共享引用
const a = { nested: { x: 1 } }
const b = Object.assign({}, a)
b.nested.x = 99 // a.nested.x 也变 99!

14. 对象展开运算符 ...ES2018

更简洁的浅拷贝和合并方式，可配合 rest 提取剩余属性

// 合并（后面的覆盖前面的同名属性）
const merged = { ...obj1, ...obj2 }
// 覆盖特定属性
const updated = { ...user, age: 26 }
// rest：提取剩余属性
const { name, ...rest } = user
→ name="张三", rest={age:25, city:"北京"}

15. Object.create — 指定原型创建对象

创建新对象并以指定对象为原型，第二个参数可定义属性描述符

// 创建一个没有原型的"纯净"对象
const pure = Object.create(null)
pure.toString // undefined! 没有继承任何方法
// 基于原型创建
const parent = { greet() { return 'hello' } }
const child = Object.create(parent)
child.greet()
→ "hello"（通过原型链调用）

16. Object.defineProperty — 属性描述符

精确控制属性的行为：可写、可枚举、可配置、getter/setter

const obj = {}
Object.defineProperty(obj, 'secret', {
  value: 42,
  writable: false, // 不可修改
  enumerable: false, // 不可枚举（for...in 看不到）
  configurable: false, // 不可删除/重新配置
})

17. getter / setter — 计算属性ES5+

像访问属性一样调用方法，内部可做计算和校验

const user = {
  firstName: '三',
  lastName: '张',
  get fullName() {
    return this.lastName + this.firstName
  },
  set fullName(val) {
    [this.lastName, this.firstName] = val.split(' ')
  }
}
user.fullName // → "张三"
user.fullName = '李 四' // 调用 setter

18. 对象锁：freeze / seal / preventExtensions

三个等级的不可变性控制

方法添加属性删除属性修改值修改描述符
preventExtensions❌✅✅✅
seal❌❌✅❌
freeze❌❌❌❌
Object.isFrozen(obj) / isSealed(obj) // 检查状态
// ⚠️ freeze 也是浅层的，嵌套对象仍可修改

19. Object.is — 严格相等ES6

比 === 更精确，解决了 NaN 和 ±0 的问题

// === 的两个"bug"
NaN === NaN
→ false（奇怪！）
0 === -0
→ true（但 0 和 -0 是不同的！）
// Object.is 都能区分
Object.is(NaN, NaN)
→ true ✅
Object.is(0, -0)
→ false ✅

20. Object.hasOwnES2022

推荐替代 obj.hasOwnProperty()，更安全（不受原型污染）

const obj = { x: 1 }
// 旧方法（可能被覆盖）
obj.hasOwnProperty('x')
// 新方法（推荐）
Object.hasOwn(obj, 'x')
→ true
// 为什么不用 hasOwnProperty：
const bad = { hasOwnProperty: false }
bad.hasOwnProperty('x') // TypeError!

21. 属性简写与计算属性名ES6

简化对象字面量的写法

// 属性简写（变量名=属性名时）
const name = '张三', age = 25
const user = { name, age }
→ { name: "张三", age: 25 }
// 方法简写
{ sayHi() { console.log('hi') } }
// 计算属性名
const key = 'dynamicKey'
const obj = { [key]: 42 }
→ { dynamicKey: 42 }

22. 可选链 ?. 与空值合并 ??ES2020

安全访问深层属性，避免 Cannot read property of undefined

// 可选链 — 遇到 null/undefined 返回 undefined
user?.address?.city
obj?.method?.() // 安全调用方法
arr?.[0] // 安全访问数组索引
// 空值合并 ?? — 仅对 null/undefined 生效
const val = input ?? '默认值'
// vs ||：|| 对 0、''、false 都会取默认值
0 || 42 // → 42（可能不是想要的）
0 ?? 42 // → 0（0 是合法值，不取默认）

23. 对象解构赋值ES6

从对象中提取多个属性到变量，支持重命名、默认值、嵌套

const { name, age } = user
// 等价于 const name = user.name; const age = user.age
// 重命名 + 默认值
const { name: userName, role = 'guest' } = user
// 嵌套解构
const { address: { city } } = user
// 函数参数解构（React/Vue 中极常用）
function UserCard({ name, age, onDelete }) { ... }
