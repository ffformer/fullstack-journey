03 — 闭包基础（Closure）

一、什么是闭包？

闭包 = 函数 + 函数定义时的词法环境（它能访问的外部变量）

当一个函数"记住"了它被创建时所在的作用域中的变量，即使这个作用域已经"销毁"，它依然能访问那些变量——这就是闭包。

示例：最简单的闭包

function createCounter() {
  let count = 0;           // ← 这个变量在 createCounter 内部

  return function() {      // ← 返回的函数"记住"了 count
    count++;               // 即使 createCounter 执行完了
    console.log(count);    // 这个函数依然能访问 count
  };
}

const counter = createCounter(); // createCounter() 执行完毕了
counter(); // 1 ← count 没有被销毁！
counter(); // 2 ← 还在累加！
counter(); // 3 ← 这就是闭包的力量

二、为什么需要闭包？

JS 中，函数每次执行完，内部的局部变量通常会被垃圾回收。但闭包让这些变量"活"了下来，因为返回的函数还引用着它们。

这给了我们三个关键能力：

  1. 私有变量 — 外部无法直接修改内部数据
  2. 状态保持 — 函数调用之间记住上一次的状态
  3. 函数工厂 — 批量生产功能相似的函数

三、闭包的三个经典应用

应用1：私有变量（数据隐藏）

function createBank() {
  let money = 1000; // 私有变量，外部无法直接访问

  return {
    getMoney() { return money; },    // 只读访问
    deposit(n)  { money += n; },     // 受控修改
    withdraw(n) {
      if (n > money) return '余额不足';
      money -= n;
      return '取款成功';
    }
  };
}
const bank = createBank();
console.log(bank.money);       // undefined — 外面拿不到
console.log(bank.getMoney());  // 1000 — 只能通过提供的方法访问

应用2：状态保持（记忆）

// 记录函数被调用了多少次
function createCallTracker() {
  let count = 0;
  return function(label) {
    count++;
    return label + ' 是第 ' + count + ' 次调用';
  };
}
const track = createCallTracker();
track('点击'); // "点击 是第 1 次调用"
track('提交'); // "提交 是第 2 次调用"
track('保存'); // "保存 是第 3 次调用"

应用3：函数工厂（批量生产）

// 根据倍率生成不同的乘法器
function createMultiplier(n) {
  return function(x) {
    return x * n; // "记住"了参数 n
  };
}
const double  = createMultiplier(2);
const triple  = createMultiplier(3);
const tenTimes = createMultiplier(10);

double(5);   // 10
triple(5);   // 15
tenTimes(5); // 50

四、经典面试题：循环中的闭包

问题：以下代码输出什么？为什么？

for (var i = 1; i <= 5; i++) {
  setTimeout(function() {
    console.log(i);
  }, i * 100);
}
// 输出：6 6 6 6 6（不是 1 2 3 4 5！）

原因：var i 是函数作用域，循环结束后只有一个 i，值是 6。所有回调共享同一个 i。

解法1：用 let 创建块作用域

for (let i = 1; i <= 5; i++) { ... } // let 每次迭代绑定新值

解法2：用闭包 + IIFE 创建独立作用域

for (var i = 1; i <= 5; i++) {
  (function(j) {           // IIFE — 立即执行函数表达式
    setTimeout(function() {
      console.log(j);      // j 是每次迭代的独立副本
    }, j * 100);
  })(i);                   // 把当前 i 传进去
}

五、闭包的潜在问题 — 内存泄漏

闭包会阻止垃圾回收，因为它引用的变量不能被释放。

如果不需要了，把引用设为 null：

let heavy = createBigClosure(); // 占用大量内存
// ... 使用完毕 ...
heavy = null; // 解除引用，让 GC 回收

提示：绝大多数场景不需要担心，只有在 DOM 引用 + 闭包组合时才容易出问题。
