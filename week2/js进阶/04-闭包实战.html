04 — 闭包实战：防抖、节流、柯里化

这三个是面试最高频的闭包应用题，必须能手写。

一、防抖（Debounce）

概念：持续触发时不执行，等停止触发后 N 毫秒才执行一次。如果又在 N 毫秒内触发，重新计时。

场景：搜索框输入（等用户打完字再发请求）、窗口 resize、按钮防连点

手写防抖源码

function debounce(fn, delay) {
  let timer = null; // 闭包变量 — 保存定时器 ID

  return function(...args) {
    // 每次调用，先清除上一次的定时器
    clearTimeout(timer);
    // 重新设置定时器
    timer = setTimeout(() => {
      fn.apply(this, args); // 执行函数，保持 this 和参数
    }, delay);
  };
}

二、节流（Throttle）

概念：持续触发时，每隔 N 毫秒固定执行一次。不管你触发多快，频率不变。

场景：滚动事件、鼠标移动、游戏帧率控制

手写节流源码（时间戳版）

function throttle(fn, delay) {
  let lastTime = 0; // 闭包变量 — 记录上次执行时间

  return function(...args) {
    const now = Date.now();
    // 距离上次执行超过 delay 毫秒，才执行
    if (now - lastTime >= delay) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}

三、柯里化（Currying）

概念：把一个多参数函数，转换成一系列单参数函数。每次只传一个参数，返回一个等待下一个参数的新函数。

场景：参数复用、延迟执行、函数组合

手写柯里化源码

// 普通函数
function add(a, b, c) { return a + b + c; }

// 柯里化版本
function curry(fn) {
  return function curried(...args) {
    // 参数够了就执行，不够就返回新函数继续收集
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    }
    return function(...args2) {
      return curried.apply(this, args.concat(args2));
    };
  };
}
const curriedAdd = curry(add);
curriedAdd(1)(2)(3); // 6
curriedAdd(1, 2)(3); // 6 — 也可以一次传多个

四、单例模式（Singleton）

概念：用闭包保证一个类只有一个实例，多次调用返回同一个对象。

场景：全局状态管理器、数据库连接、日志收集器

手写单例模式源码

function createSingleton(Constructor) {
  let instance = null; // 闭包变量 — 保存唯一实例

  return function(...args) {
    if (!instance) {
      instance = new Constructor(...args); // 第一次调用才创建
    }
    return instance; // 之后都返回同一个
  };
}

function Logger(name) {
  this.name = name;
  this.logs = [];
}
const getLogger = createSingleton(Logger);
const a = getLogger('A');
const b = getLogger('B');
console.log(a === b); // true — 同一个实例！
console.log(a.name);  // "A" — name 是第一次创建时设置的

五、闭包的注意事项

经典内存泄漏场景

// ❌ 错误：闭包引用了不再需要的 DOM 元素
function bindEvent() {
  const bigData = new Array(1000000).fill('*'); // 大对象
  const btn = document.getElementById('myBtn');

  btn.addEventListener('click', function() {
    // 这个闭包引用了 bigData，即使你只用 btn，bigData 也释放不了
    console.log(btn.textContent);
  });
  // 即使 btn 被移除，只要事件监听器还在，bigData 就占据内存
}

// ✅ 正确：用完就解除引用
function bindEventSafe() {
  let bigData = new Array(1000000).fill('*');
  const btn = document.getElementById('myBtn');

  btn.addEventListener('click', function() {
    console.log(btn.textContent);
  });

  bigData = null; // 显式解除引用
}
