错误处理 — throw · try · catch · finally

程序不可能不出错，关键是出错之后怎么处理 | 打开控制台（F12）看演示

一、什么是错误（Error）

1. 不处理错误的后果

一旦抛出错误且没被捕获，后面的代码全部不执行

console.log('① 正常执行')
throw new Error('出事了!')
console.log('② 永远不会执行') // 被错误中断
→ ① 正常执行 → 💥 程序崩溃 → ② 不执行

2. 错误的本质：一种"信号"

错误不是坏事——它是代码告诉你"这个路径走不通"的方式

// 生活类比：
// 你去银行取钱 → 卡里没钱 → ATM 提示"余额不足"
// 这个"提示"就是错误——它阻止了不该发生的事
// 如果没有这个错误，你就透支了
// 代码中的等价物：
function withdraw(balance, amount) {
  if (amount > balance) {
    throw new Error('余额不足') // 阻止透支
  }
}

3. JS 内置的错误类型

不同场景 JS 会自动抛出不同类型的错误

错误类型触发场景示例
SyntaxError语法错误const a =
ReferenceError访问未定义变量console.log(xxx)
TypeError类型不对null.name
RangeError数值超出范围[].length = -1
URIErrorURI 编解码出错decodeURIComponent('%')

4. Error 对象的三个属性

每个错误都带有这些信息，debug 的核心线索

const err = new Error('用户名不能为空')
err.name // → "Error" — 错误类型
err.message // → "用户名不能为空" — 人类可读的描述
err.stack // → 调用栈：从哪一层代码触发的
// stack 是最有价值的调试信息：
// Error: 用户名不能为空
// at validate (app.js:12:15)
// at submit (app.js:45:10)
// at HTMLButtonElement.onclick (index.html:8:5)

二、try/catch：捕获并处理错误

核心语法：用 try 包裹可能出错的代码，出错后跳到 catch 处理，程序不会崩溃

5. 基本语法

try 里放可能出错的代码，catch 里做补救

try {
  // 可能出错的代码
  const data = JSON.parse(userInput)
  console.log('解析成功:', data)
} catch (error) {
  // 出错了才到这里，不会崩溃
  console.error('JSON 格式错误:', error.message)
}
// ✅ 程序继续运行！

6. 执行流程

try 中一旦出错，立刻跳到 catch，try 剩余代码不执行

try { 开始执行
→
正常？
出错了？
↘↗
正常完成 → 跳过 catch
出错 → 跳入 catch
// ⚠️ 关键点：try 中出错位置之后的代码不会执行
try {
  console.log('A')
  throw new Error('中断!')
  console.log('B') // ❌ 不执行
} catch (e) { console.log('C') }
→ 输出：A → C（跳过了 B）

7. catch 的参数可以省略

如果你不需要用到 error 对象，可以完全不写括号（ES2019）

// 需要用到错误信息
catch (err) { console.error(err.message) }
// 不需要错误信息——ES2019 可以直接省略
catch {
  // 直接做补救工作，不需要 err
  console.log('出错了，但我不关心是什么错')
}

8. 判断错误类型——分类处理

不同错误类型用不同的处理方式，用 instanceof 判断

try {
  riskyOperation()
} catch (err) {
  if (err instanceof TypeError) {
    // 类型错误的处理方式
  } else if (err instanceof RangeError) {
    // 范围错误的处理方式
  } else {
    throw err // 不认识的错误，继续往上抛
  }
}

三、finally：无论如何都要执行

9. finally 的用途

无论 try 成功还是 catch 捕获，finally 块都会执行

try {
  // 数据库查询
} catch (err) {
  // 处理错误
finally {
  // ✅ 无论成功失败，都要关闭数据库连接
}
// 经典场景：隐藏 loading 动画
let loading = true
try { await fetch('/api/data') }
catch (err) { showError('请求失败') }
finally { loading = false } // 无论成功失败都关 loading

10. finally 的两个特殊行为

finally 的 return 会覆盖 try/catch 中的 return

// ⚠️ finally 中的 return 会"吃掉"try 的返回值
function demo() {
  try {
    return 'try 的返回值'
  } finally {
    return 'finally 覆盖了!' // 不推荐
  }
}
demo()
→ "finally 覆盖了!"（finally 的 return 赢了）

四、throw 与自定义错误

11. throw 可以抛出任何值

虽然可以 throw 任何东西，但强烈建议只 throw Error 实例

// 技术上合法但不推荐
throw '出错啦' // throw 字符串
throw 42 // throw 数字
throw { code: 500 } // throw 对象
// ✅ 推荐：永远 throw Error 实例
throw new Error('有意义的错误描述')
// 因为 Error 实例自带 stack 堆栈信息，方便定位

12. 自定义错误类实战

继承 Error 创建业务专属的错误类型，让错误更具语义

// 创建业务错误类
class ValidationError extends Error {
  constructor(message, field) {
    super(message)
    this.name = 'ValidationError'
    this.field = field // 哪个字段出错了
  }
}
// 使用自定义错误
if (!email.includes('@')) {
  throw new ValidationError('邮箱格式不正确', 'email')
}
// catch 时可以精确判断：
if (err instanceof ValidationError) { ... }

五、实战模式

13. 模式一：安全 JSON 解析

用户输入/API 响应的 JSON 可能格式错误，必须 try/catch

function safeJsonParse(str, fallback = null) {
  try {
    return JSON.parse(str)
  } catch {
    return fallback // 解析失败返回默认值
  }
}

14. 模式二：表单验证

收集所有验证错误，统一抛出

function validate(form) {
  const errors = []
  if (!form.name) errors.push('姓名必填')
  if (!form.email.includes('@')) errors.push('邮箱格式错误')
  if (errors.length) throw new Error(errors.join('；'))
}

15. 模式三：优雅降级

某个功能崩了不应影响整体页面——前端重要原则

try {
  // 尝试加载第三方组件（可能因为 CDN 挂掉失败）
  initChart()
} catch {
  // 降级：显示一个"图表加载失败"的占位图
  showFallbackPlaceholder()
}
→ 图表坏了，但页面其他功能正常使用

16. Promise 的 .catch() 对标 try/catch

理解了 try/catch，再看 Promise 的错误处理是一回事

// 同步错误处理
try { riskySync() }
catch (err) { /* 处理 */ }
// 异步错误处理（Promise 版）
riskyAsync()
  .then(result => /* 成功 */)
  .catch(err => /* 处理 — 对标 try/catch */)
  .finally(() => /* 清理 — 和 finally 一样 */)
// 异步错误处理（async/await 版 — 直接就是 try/catch!）
async function main() {
  try { await riskyAsync() }
  catch (err) { /* 处理 */ }
}
