箭头函数深度解读

语法糖只是表面，核心是：没有自己的 this、arguments、prototype

一、基础语法

1. 语法变体

从最简到最完整的四种写法

// ① 单参数 + 单表达式（最简）
x => x * 2
// ② 多参数需要括号
(a, b) => a + b
// ③ 函数体多行需要 {} + return
(a, b) => {
  const sum = a + b
  return sum * 2
}
// ④ 返回对象字面量需要括号包起来
(name, age) => ({ name, age })
// 不加括号 {} 会被当成函数体！

2. 隐式返回 vs 显式返回

省略 {} 时自动返回表达式的值（也叫 implicit return）

// 隐式返回：省略 {} 和 return
const double = n => n * 2
double(5)
→ 10
// 显式返回：写了 {} 就必须写 return
const double2 = n => { return n * 2 }
// ⚠️ 常见错误：写了 {} 但没写 return
const bad = n => { n * 2 }
bad(5)
→ undefined（花括号里需要 return！）

3. 参数规则与默认值

支持默认参数、剩余参数，但不支持 arguments 对象

// 默认参数
const greet = (name = '匿名') => `你好，${name}`
// 剩余参数（替代 arguments）
const sum = (...nums) => nums.reduce((s, n) => s + n, 0)
sum(1, 2, 3, 4)
→ 10
// 带默认值的解构参数
const print = ({ name, age = 0 }) => `${name}: ${age}`

4. 立即执行箭头函数（IIFE）

箭头函数也能立即执行，语法比传统 IIFE 更简洁

// 传统函数 IIFE
(function() { console.log('ok') })()
// 箭头函数 IIFE（注意括号位置）
(() => {
  const secret = '局部变量'
  console.log(secret)
})()

二、核心特性：this 绑定（最重要）

箭头函数没有自己的 this → 继承外层作用域的 this → 这就是"词法 this"

5. this 对比：普通函数 vs 箭头函数核心

普通函数的 this 看调用者；箭头函数的 this 看定义时外层作用域

const obj = {
  name: '张三',
  // 普通函数：this = 调用时 . 前面的对象
  sayName1() { console.log(this.name) },
  // 普通函数简写（ES6 方法简写，等价上面）
  sayName2: () => console.log(this.name),
}
obj.sayName1() // → "张三"
obj.sayName2() // → undefined（this 是外层/window！）

6. 经典场景：用箭头函数解决 this 丢失核心

回调函数中 this 会丢失，箭头函数天然解决——不需要 that=this 了

// ❌ 普通函数写法：this 丢失
class Counter {
  constructor() { this.count = 0 }
  start() {
    setInterval(function() {
      this.count++ // ❌ this 指向 window！
    }, 1000)
  }
}
// ✅ 箭头函数：this 继承自 start() 的 this
class Counter {
  constructor() { this.count = 0 }
  start() {
    setInterval(() => {
      this.count++ // ✅ this = 实例
    }, 1000)
  }
}

7. 箭头函数无法用 call/apply/bind 改变 this

箭头函数的 this 在定义时就固定了，call/apply/bind 传参会被忽略

const fn = () => console.log(this)
fn.call({ x: 1 })
→ Window（不是 {x:1}！this 不变！）
// 但 call/apply 仍然可以传参数
const add = (a, b) => a + b
add.call(null, 1, 2)
→ 3（参数正常传入，只是 this 不生效）

8. 箭头函数嵌套：this 穿透

无论嵌套多深，箭头函数的 this 始终指向最外层的普通函数 this

const obj = {
  name: '外层',
  outer() {
    const inner = () => {
      const deep = () => {
        console.log(this.name) // 穿透两层，指向 obj
      }
      return deep()
    }
    return inner()
  }
}
obj.outer()
→ "外层"（this 穿透两层箭头函数）

三、箭头函数不能做的事（限制）

9. 不能用作构造函数（不能用 new）

箭头函数没有 [[Construct]] 内部方法，new 会直接报错

const Foo = () => {}
new Foo()
→ TypeError: Foo is not a constructor
// 为什么？箭头函数没有 prototype 属性
const fn = () => {}
fn.prototype
→ undefined

10. 没有 arguments 对象

访问 arguments 会沿作用域链查找外层的 arguments

const fn = () => {
  console.log(arguments)
}
fn(1, 2, 3)
→ 浏览器里是外层 arguments / Node 里直接报错
// ✅ 用剩余参数替代
const fn2 = (...args) => console.log(args)
fn2(1, 2, 3)
→ [1, 2, 3]

11. 不能用作生成器函数

箭头函数内部不能用 yield，也无法用 function* 定义

// ❌ 语法错误
const gen = () * => { ... } // 不存在这种写法
// ✅ 生成器必须用普通函数
function* gen() { yield 1; yield 2 }

12. 没有 new.target

在箭头函数里访问 new.target 会取外层函数的值

function Foo() {
  const inner = () => console.log(new.target)
  inner() // 继承 Foo 的 new.target
}
new Foo()
→ [Function: Foo]

四、什么时候不该用箭头函数

13. ❌ 对象的方法

对象方法需要 this 指向对象本身，箭头函数做不到

// ❌ 错误
const user = {
  name: '张三',
  sayName: () => console.log(this.name)
}
// ✅ 正确：方法简写（ES6）
const user = {
  name: '张三',
  sayName() { console.log(this.name) }
}

14. ❌ 原型上的方法

和对象方法同理——需要 this 指向实例

// ❌ 错误
function User(name) { this.name = name }
User.prototype.sayName = () => console.log(this.name)
// ✅ 正确：普通函数
User.prototype.sayName = function() { console.log(this.name) }

15. ❌ 需要 this 指向 DOM 元素的事件

如果想在事件中通过 this 获取 DOM 元素，不能用箭头函数

// ❌ this 不指向按钮
btn.addEventListener('click', () => console.log(this))
→ Window（不是 btn）
// ✅ 用 event.target 替代（推荐方式）
btn.addEventListener('click', (e) => console.log(e.target))
→ btn 元素（用 e.target，this 不再重要）
// ✅ 或坚持用 this，那就用普通函数
btn.addEventListener('click', function() { console.log(this) })

16. ❌ 需要动态 this 的场景

call/apply/bind 无法改变箭头函数的 this

// 需要借用方法时，不能用箭头函数
// 比如：Array.prototype.slice.call(arguments)
// 这种依赖 this 动态绑定的场景必须用普通函数

六、箭头函数实战模式

17. 数组处理流水线常用

配合 map/filter/reduce 实现链式数据处理

const result = [1, 2, 3, 4, 5]
  .filter(n => n % 2 === 0)
  .map(n => n * n)
  .reduce((sum, n) => sum + n, 0)
→ 20（偶数 2,4 → 平方 4,16 → 求和 20）

18. Promise / 异步回调常用

箭头函数让 .then().catch() 链变得极简

fetch('/api/user')
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err))
// async/await 中的辅助箭头函数
const userIds = users.map(u => u.id)
const names = users.filter(u => u.active).map(u => u.name)

19. 组件中的箭头函数框架

在 React 类组件中箭头函数自动绑定 this；Vue 中常用于 methods

// React 类组件：箭头函数自动绑定
class Button extends React.Component {
  handleClick = () => {
    this.setState({ ... }) // ✅ this 自动正确
  }
}
// Vue 3 composition API：箭头函数或普通函数都行
const handleClick = () => { state.count++ }
function handleClick() { state.count++ } // 这个也行

20. 函数组合 / 柯里化

箭头函数天然适合表达式式的函数组合风格

// 柯里化：用箭头函数的链式写法
const add = a => b => a + b
const add5 = add(5)
add5(3)
→ 8
// 函数组合（compose）
const compose = (f, g) => x => f(g(x))
const upper = s => s.toUpperCase()
const greet = s => `你好，${s}`
const shout = compose(upper, greet)
shout('张三')
→ "你好，张三"
